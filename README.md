# Commons Preservation License (CPL) v1.0

A share-alike license for keeping knowledge, code, and AI-derived systems open and freely accessible.

## What is CPL?

The Commons Preservation License (CPL) is designed for people who want to share useful work freely while preserving the same freedoms for everyone downstream.

CPL allows people to:
- Use the work
- Learn from it
- Modify it
- Share it
- Train AI on it

CPL requires that:
- Derivative works remain under CPL
- Source code remains publicly available
- AI models trained on CPL-covered work remain open
- The work is not enclosed behind proprietary or paid access barriers

## How to use CPL in your project

1. Copy one of the license files from this repository into your project:
   - `LICENSE`
   - `LICENSE.md`
   - `LICENSE.txt`
2. Replace the placeholders:
   - `[YEAR]` with the current year
   - `[AUTHOR]` with your name, handle, or organization
3. Keep the full license text intact.
4. Optionally reference this repository in your project README.

Example:

```md
Licensed under the [Commons Preservation License (CPL) v1.0](https://github.com/[YOUR-USERNAME]/CPL)
```

## Files in this repository

- `LICENSE` — standard plain license file
- `LICENSE.md` — Markdown-rendered version for GitHub
- `LICENSE.txt` — plain text version

## Why CPL exists

Knowledge should be free. Tools should serve people, not profits.

CPL is meant to protect shared work from enclosure while still encouraging learning, experimentation, and original innovation. It is not meant to discourage people from creating new things or benefiting from their own contributions. It is meant to ensure that if someone builds directly on CPL-covered work, the same freedoms are preserved for others.

## Short reference text

Projects using CPL may use the following short notice:

